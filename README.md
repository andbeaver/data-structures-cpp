# Data Structures Assignments

This repository is a collection of separate data structures assignments. Each assignment lives in its own folder, with its own source files, CMake build file, and assignment-specific instructions.

## Repository layout

- `Binary Search Trees/` — Assignment 3. A self-contained project for the spell checker / balanced tree assignment.
  - `src/` — C++ source and header files for the assignment.
  - `tests/` — Input files used to test the program.
  - `output/` — Generated output files created by the program.
  - `docs/` — Assignment PDFs and supporting documents.
  - `README.md` — Assignment-specific instructions.

- `Linked Lists/` — Assignment 1. A self-contained project focused on linked list operations.
  - `src/` — C++ source and header files for the assignment.
  - `README.md` — Assignment-specific instructions and test procedure.

- `Sorting/` — Assignment 4. A self-contained project for sorting employee records with merge sort.
  - `src/` — C++ source and header files for the assignment.
  - `output/` — Generated sorted output files.
  - `employees.txt` — Sample input data used by the assignment.
  - `docs/` — Assignment PDFs and rubric files.
  - `README.md` — Assignment-specific instructions.

- `Stacks and Queues/` — Assignment 2. A self-contained project for the stacks and queues / maze assignment.
  - `src/` — C++ source and header files for the assignment.
  - `tests/` — Test mazes and input files.
  - `solved/` — Saved solved output files.
  - `docs/` — Assignment PDFs and supporting documents.
  - `README.md` — Assignment-specific instructions and test procedure.

## How to use this repository

1. Open the specific assignment folder you want to work on in CLion.
2. Build and run that assignment from inside its own folder.
3. Read the assignment-specific `README.md` in that folder before starting.
4. Keep generated files such as build folders, output files, and temporary files within the assignment they belong to.

## Notes

- Each assignment is intentionally independent, so they can be developed and submitted separately.
- If you add new assignments later, follow the same pattern: one folder per assignment with its own `src/`, `docs/`, and `README.md` as needed.
